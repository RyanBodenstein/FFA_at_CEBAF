SECTION 6: Impact on Intra-Plate NdFeB−SmCo Differential
================================================================================

The intra-plate differential measures NdFeB−SmCo on the SAME plate
in the SAME session. If gain is material-independent, the differential
is immune to gain shifts. But if gain is material-dependent...

  2025-04-23: NdFeB−SmCo diff = -0.0414%
  2025-05-07: NdFeB−SmCo diff = -0.8374%
  2025-05-21: NdFeB−SmCo diff = -0.6287%
  2025-06-11: NdFeB−SmCo diff = -0.6940%
  2025-06-17: NdFeB−SmCo diff = +0.2511%

  KEY QUESTION: In the actual TUNNEL measurements, are NdFeB and SmCo
  on the same plate measured at exactly the same temperature?
  YES — they're on the SAME plate, measured within minutes of each other.
  The gain (electronics drift) affects both identically.
  But TEMPERATURE affects them differently (different α values).

  In the pre-deployment lab sessions, temperature varies BETWEEN sessions
  but NdFeB and SmCo on the same plate experience the SAME temperature.
  So the NdFeB−SmCo differential within a session reflects:
    (1) True gain difference between materials (should be zero)
    (2) Temperature × Δα effect (real, but cancels in differential
        because both are at the same temp in the same session)

  WAIT — let's be more careful. The gain function computes:
    pct = (check_date_mWC − ref_date_mWC) / ref_date_mWC × 100
  This compares the SAME sample at two DIFFERENT dates.
  If temps differ between dates, NdFeB shifts more than SmCo.
  But the INTRA-PLATE DIFFERENTIAL computes:
    diff = (NdFeB_pct_change) − (SmCo_pct_change)
  where each pct_change = (tunnel − baseline) / baseline × 100
  Both measured in the same session → same gain multiplier.
  The differential is gain-immune REGARDLESS of material dependence
  in the gain estimate, because the gain multiplier cancels out.

  HOWEVER: if temperature differs between baseline and tunnel sessions,
  the NdFeB pct_change includes a temperature component that differs
  from the SmCo temperature component (different α).
  This is EXACTLY what the temperature correction addresses.
  After temp correction, the differential should be clean.

  ✓ Tunnel Y-plate analysis IS temperature-corrected
    (manager_summary_v3.py applies per-sample temp correction)
  ✓ The intra-plate differential uses temp-corrected % changes
  ✓ Therefore the differential is BOTH gain-immune AND temp-corrected
  ✓ The ±0.248% gain systematic does NOT apply to the differential

  CONCLUSION: The gain systematic estimate itself conflates gain + temperature,
  but this doesn't matter for the headline 9.7σ result because the
  intra-plate differential cancels gain exactly (same session, same electronics)
  and the temperature correction handles the α difference.

SECTION 7: Blanket Uncertainty vs Per-Session Correction
================================================================================

  CURRENT APPROACH:
  - ±0.248% applied as uncertainty band on ALL absolute % changes
  - No correction applied to raw data
  - Intra-plate differential claimed to be gain-immune (correct)

  ALTERNATIVE: Per-Session Normalization
  - Use SmCo mean as session reference (SmCo shouldn't degrade much)
  - Or use a dedicated calibration sample (user plans to add this)
  - Normalize each session's readings to the reference
  - Would reduce uncertainty from ±0.25% to ±(SmCo scatter / √N)

  PROS of per-session normalization:
  + Removes systematic gain offset from absolute values
  + Could make H-plate and A-sample results more precise
  + Would allow H/A single-material plates to contribute meaningfully

  CONS of per-session normalization:
  - SmCo is not a perfect reference (−0.08% possible degradation)
  - Only works if gain is truly material-independent
  - Requires enough reference samples per session for statistics
  - CURRENTLY IMPOSSIBLE: the 5 pre-deployment sessions have NO temperature
    data, so can't separate gain from temperature

  ASSESSMENT:
  The current approach (blanket uncertainty band) is CONSERVATIVE and CORRECT
  for the following reasons:
  1. The headline result (9.7σ intra-plate differential) doesn't use the gain
     systematic at all — it's gain-immune by construction
  2. For absolute values (H-plates, A-samples), the ±0.25% band correctly
     reflects that we CANNOT distinguish gain from temperature effects
  3. Per-session normalization would require either:
     (a) Temperature data for all sessions (unavailable for pre-deployment)
     (b) A dedicated calibration sample (planned for future runs)
  4. Using SmCo as a reference would introduce a circular argument
     (assumes SmCo doesn't degrade to measure if NdFeB degrades)

  HOWEVER, the gain estimate itself (±0.248%) has issues:
  1. It conflates gain drift with temperature drift
  2. It uses only 5 sessions, all from a 2-month window
  3. The half-range estimator is crude (range / 2 with N=5)
  4. All 5 sessions show NEGATIVE offset from Nov 2024, suggesting
     a single systematic shift rather than random session-to-session noise

  RECOMMENDATION:
  - Keep the current approach (uncertainty band, not correction)
  - Acknowledge that ±0.248% likely OVERESTIMATES the true gain-only uncertainty
    (it includes temperature drift)
  - The true gain-only uncertainty is probably ±0.10-0.15% (temperature
    accounts for the material-dependent part ~0.10-0.15%)
  - For the headline result (intra-plate differential), this doesn't matter
  - For H/A absolute results, the current band is conservative (good)
  - Future: dedicated cal samples + lab temperature logging will resolve this
